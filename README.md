# RTT Tennis Match Predictor

Учебный ML-проект для прогнозирования вероятности исхода одиночных матчей Российского теннисного тура (РТТ). Основной фокус проекта - матчи женского одиночного разряда, в том числе возрастные категории до 17 лет, до 19 лет и взрослые турниры.

Проект не является коммерческим сервисом, не предназначен для ставок и не претендует на официальную аналитику РТТ. Данные получены из публично доступных источников РТТ и используются в учебных целях.

## Что Делает Проект

Проект собирает данные с сайта РТТ, формирует обучающий датасет, строит признаки по игрокам и матчам, сравнивает несколько моделей машинного обучения и дает прогноз по конкретной паре игроков.

Полный сценарий:

1. Загрузить список турниров из календаря РТТ.
2. Получить детали турниров и страницы матчей.
3. Скачать и распарсить рейтинги игроков на доступные даты.
4. Собрать финальный датасет для модели.
5. Обучить и сравнить модели.
6. Переобучить выбранную production-модель на всей доступной истории.
7. Получить прогноз конкретного матча и аналитические блоки по игрокам.

## Качество Моделей

Модели сравниваются на временном test split. Test-период всегда равен последним 3 месяцам от максимальной даты матча в актуальном датасете. Основной критерий выбора - test LogLoss на match-level symmetrized прогнозе.

| Модель | LogLoss | Brier | ROC-AUC | Accuracy |
| --- | ---: | ---: | ---: | ---: |
| Gradient Boosting | 0.4557 | 0.1498 | 0.8658 | 0.7810 |
| CatBoost | 0.4561 | 0.1501 | 0.8653 | 0.7826 |
| Random Forest | 0.4621 | 0.1518 | 0.8621 | 0.7721 |
| Rating + ELO baseline | 0.4942 | 0.1635 | 0.8403 | 0.7575 |
| ELO only | 0.6003 | 0.2065 | 0.7475 | 0.6918 |

В последнем прогоне лучшей моделью по LogLoss стал `GradientBoostingClassifier`. Подробнее см. [docs/model_report.md](docs/model_report.md).

## Основные Признаки

В модели используются:

- официальный рейтинг РТТ и рейтинговые очки до матча;
- скорректированный рейтинг: очки на один зачетный турнир;
- относительные разницы рейтинговых показателей;
- ELO и ожидаемая вероятность победы по ELO;
- личные встречи до даты матча;
- общая история игрока и недавняя форма;
- отдых с последнего матча;
- сила недавних соперников;
- результаты против общих соперников;
- индикаторы пропусков рейтинговых данных.

Исторические признаки считаются хронологически: для матча используются только данные, доступные до его даты.

## Единая Контрольная Панель

Основная рабочая точка проекта:

```text
notebooks/00_data_control_panel.ipynb
```

В панели доступны:

- проверка свежести локальных данных;
- обновление календаря турниров РТТ;
- запуск полного пайплайна обновления и обучения;
- прогноз конкретного матча;
- карточки двух игроков на дату предполагаемого матча;
- историческая динамика рейтинга, рейтинговых очков, ELO и формы;
- последние 10 матчей каждого игрока;
- основные факторы, влияющие на итоговую вероятность;
- история вероятности победы игрока над выбранным соперником;
- личные встречи и подробный анализ игры против общих соперников.

## Скрипты

Основной воспроизводимый путь сейчас script-first:

```bash
python scripts/parse_rtt_calendar.py
python scripts/parse_rtt_matches.py
python scripts/parse_rtt_rankings.py
python scripts/build_final_dataset.py
python scripts/train_model.py
```

Полная цепочка запускается одной командой:

```bash
python scripts/run_full_pipeline.py
```

Если календарь РТТ временно недоступен, можно продолжить пайплайн на уже сохраненном master-файле турниров:

```bash
python scripts/run_full_pipeline.py --continue-on-calendar-error
```

## Данные

Ключевые файлы данных:

- `data/tournaments_master.xlsx` - мастер-список турниров, используется парсером матчей;
- `data/data_manifest.json` - статусный снимок свежести данных;
- `assembled_predictor/predictor_model_dataset_from_parsers.xlsx` - финальный датасет для обучения модели.

Обновить статус данных можно так:

```bash
python scripts/data_status.py --write-manifest
```

## Установка

Минимальные зависимости для production-скриптов:

```bash
pip install -r requirements.txt
playwright install
```

Если нужно запускать ноутбуки в свежем окружении:

```bash
pip install -r requirements-notebook.txt
```

## Быстрая Проверка

Перед коммитом или после изменения структуры проекта:

```bash
python scripts/verify_project.py
```

Проверка валидирует наличие ключевых файлов, JSON-структуру ноутбуков и основные листы финального Excel-датасета. Она специально легкая и не запускает полный парсинг или обучение.

## Ограничения

- Парсинг зависит от текущей верстки сайта РТТ.
- Модель не учитывает травмы, покрытие, усталость вне истории матчей, поездки и другие внешние факторы.
- Метрики относятся к историческому test split и не гарантируют качество на будущих данных.
- Проект хранит только актуальную версию кода и документов; старые промежуточные версии ноутбуков и записок удалены из текущей ветки.
