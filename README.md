# RTT Tennis Match Predictor

Учебный ML-проект для предсказания победителя одиночного теннисного матча Российского теннисного тура (РТТ) по историческим матчам, рейтингам игроков и производным признакам формы.

Проект не является коммерческим сервисом, не предназначен для ставок и не претендует на официальную аналитику РТТ. Данные получены из публично доступных источников РТТ и использованы исключительно в учебных целях в рамках дипломного проекта по курсу «Машинное обучение в Python».

## Что делает проект

Пайплайн собирает страницы турниров и рейтинги РТТ, парсит результаты матчей, сопоставляет игроков с РНИ, подтягивает рейтинги до даты матча, строит обучающий датасет и сравнивает несколько моделей бинарной классификации.

Финальная задача модели: для пары игроков оценить вероятность победы выбранного игрока в матче.

## Ноутбуки

0. `notebooks/00_data_control_panel.ipynb`  
   Показывает локальный статус данных, обновляет `data/tournaments_master.xlsx` из текущего Excel со ссылками и запускает быструю проверку проекта.

1. `notebooks/01_save_and_parse_matches.ipynb`  
   Сохраняет страницы турниров РТТ и извлекает строки матчей из HTML.

2. `notebooks/02_parse_rankings.ipynb`  
   Парсит рейтинги игроков по датам и возрастным группам, формирует историю рейтингов и маппинг ФИО/РНИ.

3. `notebooks/03_build_final_dataset.ipynb`  
   Собирает финальный Excel-датасет: матчи, игроки, рейтинги до даты матча, признаки счета, аудит покрытия.

4. `notebooks/04_train_final_model.ipynb`  
   Строит признаки, обучает и сравнивает модели, сохраняет метрики, важности признаков и диагностические таблицы.

Важно: ноутбуки рассчитаны на запуск из корня репозитория. Относительные пути внутри проекта используют корневые папки вроде `assembled_predictor/`, `saved_rtt_pages/` и `rtt_rankings_saved/`.

## Данные и признаки

Финальный датасет хранится в `assembled_predictor/predictor_model_dataset_from_parsers.xlsx`.

Список турниров теперь хранится отдельно в `data/tournaments_master.xlsx`. Это мастер-файл для скачивания страниц матчей: новые турниры должны добавляться в него с дедупликацией по `tour_id`, а затем ноутбук `notebooks/01_save_and_parse_matches.ipynb` использует уже этот файл как источник ссылок.

Для контроля свежести локальных данных можно запустить:

```bash
python scripts/data_status.py --write-manifest
```

Команда показывает количество турниров, сохраненных HTML-страниц, свежесть рейтингов и максимальную дату матчей в финальном датасете. Снимок статуса сохраняется в `data/data_manifest.json`.

Чтобы подтянуть новые турниры из календаря РТТ и слить их в мастер-файл:

```bash
python scripts/parse_rtt_calendar.py --date-from 17.04.2026 --date-to 11.05.2026
```

По умолчанию парсер идет по фильтрам: женский одиночный разряд, возраст `до 17 лет`, `до 19 лет`, `Взрослые`, Центральный ФО. Новые строки добавляются только если `tour_id` еще нет в `data/tournaments_master.xlsx`.

Полная цепочка обновления запускается так:

```bash
python scripts/run_full_pipeline.py
```

Она обновляет список турниров и детали, запускает парсинг матчей, обновляет рейтинги, пересобирает финальный датасет и выполняет обучение. В финальном обучении test-период теперь определяется динамически: последние 3 месяца от максимальной даты матча в актуальном датасете.

В модели используются группы признаков:

- официальный рейтинг РТТ до матча;
- скорректированные рейтинговые очки на зачетный турнир;
- ELO до матча;
- личные встречи до текущего матча;
- недавняя форма и опыт игрока;
- сила предыдущих соперников;
- результаты против общих соперников;
- признаки пропусков рейтинга.

Расчет исторических признаков сделан в хронологическом порядке: для текущего матча используются только сведения, доступные до него.

## Результаты

Финальный временной test split содержит 1902 матча. Лучший результат в последнем сохраненном прогоне показал `GradientBoostingClassifier`.

| Модель | LogLoss | Brier | ROC-AUC | Accuracy |
| --- | ---: | ---: | ---: | ---: |
| Gradient Boosting | 0.4164 | 0.1337 | 0.8911 | 0.8139 |
| Random Forest | 0.4215 | 0.1353 | 0.8907 | 0.8086 |
| CatBoost | 0.4306 | 0.1376 | 0.8887 | 0.8091 |
| Rating + ELO baseline | 0.4449 | 0.1435 | 0.8749 | 0.7923 |
| ELO only | 0.5878 | 0.2010 | 0.7653 | 0.7003 |

Подробнее: `docs/model_report.md`.


## Воспроизведение

Ожидаемый порядок запуска:

0. Открыть `notebooks/00_data_control_panel.ipynb`, проверить свежесть данных и при необходимости обновить `data/tournaments_master.xlsx`.
1. Запустить `notebooks/01_save_and_parse_matches.ipynb`.
2. Запустить `notebooks/02_parse_rankings.ipynb`.
3. Запустить `notebooks/03_build_final_dataset.ipynb`.
4. Запустить `notebooks/04_train_final_model.ipynb`.

Если сырые страницы и рейтинги уже сохранены локально, можно начинать с готового датасета `assembled_predictor/predictor_model_dataset_from_parsers.xlsx` и запускать только финальный ноутбук обучения.

В финальном ноутбуке test split используется для честной оценки качества. Test-период всегда равен последним 3 месяцам от максимальной даты матча в актуальном датасете. После выбора лучшего алгоритма интерактивный предиктор переобучает такой же тип модели на всей доступной истории и использует эту `production_main_model` для пользовательских прогнозов.

## Ограничения

- Парсинг зависит от текущей верстки сайта РТТ.
- Проект не учитывает травмы, покрытие, усталость вне истории матчей и другие внешние факторы.
- Метрики относятся к историческому test split и не гарантируют качество на будущих данных.
- Репозиторий хранит только актуальную версию основного predictor-ноутбука; промежуточные версии удалены из текущей ветки.
## Production scripts

The reproducible update path is now script-first:

```bash
python scripts/parse_rtt_calendar.py
python scripts/parse_rtt_matches.py
python scripts/parse_rtt_rankings.py
python scripts/build_final_dataset.py
python scripts/train_model.py
```

`notebooks/01_save_and_parse_matches.ipynb` through `notebooks/04_train_final_model.ipynb` are kept for review, manual diagnostics, and the interactive prediction UI. `scripts/run_full_pipeline.py` runs the production scripts directly instead of executing notebooks through `nbconvert`.
